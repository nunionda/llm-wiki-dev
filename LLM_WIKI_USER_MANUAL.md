# LLM-Wiki: 글로벌 지식 허브 사용 매뉴얼

**LLM-Wiki**는 파편화된 로컬 개발 환경의 지식을 하나로 통합하여, 당신만의 거대한 '개인용 AI 뇌'를 구축하는 시스템입니다. 이 매뉴얼은 시스템의 핵심 기능과 효율적인 활용 방법을 설명합니다.

---

## 1. 개요 (Overview)

기존의 지식 관리가 프로젝트별로 고립되어 있었다면, **LLM-Wiki**는 모든 프로젝트의 마크다운 문서, 코드 주석, 리서치 노트를 중앙(`~/.LLM-Wiki`)에서 관리하고 상호 연결합니다.

*   **중앙 집중형**: 모든 프로젝트의 지식이 하나의 인덱스로 통합됩니다.
*   **시맨틱 검색**: `qmd` 엔진을 통해 키워드가 아닌 '의미' 단위로 지식을 소환합니다.
*   **멀티 워크스페이스**: 여러 프로젝트를 넘나들며 지식을 교차 참조합니다.

---

## 2. CLI 명령어 가이드 (CLI Usage)

터미널 어디에서든 `llm-wiki` 명령어를 사용하여 지능형 허브를 제어할 수 있습니다.

### `llm-wiki track .`
현재 작업 중인 폴더를 지식 허브의 워크스페이스로 등록합니다.
*   연동 시 폴더 내의 `wiki/` 디렉토리가 자동으로 탐지됩니다.
*   레지스트리(`projects.json`)에 경로가 영구 저장됩니다.

### `llm-wiki sync`
등록된 모든 워크스페이스의 변화를 감지하여 지식 그래프를 최신화합니다.
*   **매니페스트 갱신**: UI에서 볼 수 있는 파일 목록을 업데이트합니다.
*   **시맨틱 인덱싱**: 새롭게 추가된 내용에 대해 `qmd` 임베딩을 생성합니다.
*   **심볼릭 링크**: 브라우저가 각 프로젝트 폴더에 안전하게 접근할 수 있도록 경로를 연결합니다.

## 3. 통합 위키 UI 활용 (UI Interface)

서버 주소: [http://localhost:10000](http://localhost:10000)

### 지식 시맨틱 매트릭스 (Knowledge Semantic Matrix)
그래프 인터페이스 상단의 'Maximize' 버튼을 클릭하여 **풀스크린 시맨틱 매트릭스** 모드로 진입합니다.
*   **지식 연결 밀도**: 연결이 많은 '허브' 노드는 더 크게 표시되며, 시각적 광채(Glow) 효과가 적용됩니다.
*   **시맨틱 동기화 (Sync Matrix)**: 'Sync Semantic Hub' 버튼을 통해 즉시 지식 베이스를 재인덱싱하고 테마 밀도를 분석합니다.
*   **모듈별 필터링**: 인프라, 연구, 개인 등 카테고리별로 노드를 구분하여 탐색할 수 있습니다.

---

## 4. 지식 합성 동기화 (Semantic Synthesis Sync)

시스템의 정합성을 유지하기 위해 UI 내의 'Audit Matrix' 혹은 'Sync Semantic Hub' 버튼을 활용하십시오. 이 기능은 백엔드에서 다음의 자율 에이전트들을 순차적으로 실행합니다:

1.  **Bookkeeper**: 모든 문서의 변화를 감지하고 `index.md`와 관계망(`relations.json`)을 갱신합니다.
2.  **Insight Engine**: 전체 지식 노드의 테마 밀도를 스캔하여 **Global Strategy Matrix** 문서를 생성하고 '지식 격차'를 보고합니다.
3.  **Link Validator**: 모든 교차 참조([[link]])의 유효성을 검증하고 깨진 링크를 리포트합니다.

---

## 5. 멀티 워크스페이스 관리 (Multi-Workspace Workflow)

추천하는 작업 흐름은 다음과 같습니다:

1.  **등록**: 새로운 리서치나 개발을 시작할 때 해당 폴더에서 `llm-wiki track .` 실행.
2.  **작성**: 클로드와 함께 마크다운 문서를 작성 (예: `wiki/pages/` 아래).
3.  **연결**: 문서 내에서 `[[다른-프로젝트-문서-ID]]` 형식으로 교차 참조 작성.
4.  **싱크**: `llm-wiki sync`를 통해 전역 그래프에 반영.
5.  **소환**: `http://localhost:10000`에서 통합된 관점의 리포트를 확인.

---

## 5. 고급 설정 (Advanced)

*   **인프라 위치**: 모든 중앙 데이터는 `~/.LLM-Wiki/`에 저장됩니다.
*   **프로젝트 관리**: `~/.LLM-Wiki/projects.json`에서 직접 경로를 수정하거나 삭제할 수 있습니다.
*   **검색 엔진**: `qmd` 설정과 인덱스 데이터는 시스템 캐시 영역에 안전하게 보존됩니다.
